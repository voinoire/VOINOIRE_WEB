# AGENTS.md — цифровой профиль

Инструкции для AI-агента при работе с этим репозиторием.

## Суть проекта

**Цифровой профиль** — одностраничный статический сайт (landing): личная **диджитал-витрина** для обычного человека (не разработчика). Одна ссылка вместо десяти профилей в соцсетях.

**Приоритеты v1 (по убыванию):**

1. **Визуал** — современный минимализм, карточки, hover, «как в приложении»
2. **Цифровой профиль** — текст о том, кто человек в интернете
3. **Хаб ссылок** — карточки Telegram, VK, YouTube и т.д.
4. **Один проект** — одна крупная карточка, не портфолио из кейсов

**Аудитория:** друзья, подписчики из соцсетей; mobile first; без IT-сленга в UI.

**Языки:** RU (главная) и EN (`/en/`).

## Стек и ограничения

| Используем | Не используем в v1 |
|------------|-------------------|
| HTML, CSS в `<style>` в шаблонах | JavaScript, React, Node runtime |
| Python + Jinja2 (`build.py`) | Отдельные `.css` / `.js`, CMS, Vite, 11ty |
| Статика в `dist/` (Режим A) | Flask, формы, бэкенд |

**Prisma / PostgreSQL** — инициализированы (`prisma/`, `package.json`), но **не часть v1**. Статический сайт не зависит от БД. Не добавлять Prisma в сборку или шаблоны без явного запроса.

**Контент** — тексты правятся **вручную** в `templates/index.ru.html` и `templates/index.en.html`. Без YAML, CMS и автоподстановки.

## Структура репозитория

```
templates/
  base.html           # layout, <style>, header, footer, RU|EN switch
  index.ru.html       # русский контент
  index.en.html       # английский контент
build.py              # Jinja2 → dist/
assets/images/        # исходники картинок (копируются в dist/assets/)
dist/                 # сборка для публикации (в .gitignore)
план-цифровой-профиль.md   # полный план, этапы, решения
wireframe.md               # wireframe и порядок секций
контент-чеклист.md         # что подготовить для контента
```

## Целевая структура страницы

Одна landing с якорями (см. `wireframe.md`):

| Секция | id | Содержание |
|--------|-----|------------|
| Hero | `#top` | имя, аватар, слоган |
| Цифровой профиль | `#profile` | 100–300 слов о диджитал-образе |
| Интернет-профили | `#links` | grid карточек-ссылок |
| Мой проект | `#project` | **одна** карточка с превью и CTA |
| Контакты | `#contact` | email, мессенджеры |
| Footer | — | ©, опционально resume.pdf |

Header: sticky, меню по якорям, переключатель RU | EN, mobile menu на `<details>`.

## Команды

```bash
# зависимости Python
pip install -r requirements.txt

# сборка
python build.py

# локальный просмотр
python -m http.server --directory dist
# RU: http://localhost:8000/
# EN: http://localhost:8000/en/
```

Prisma (только если явно нужна БД):

```bash
npm install
npx prisma validate
```

## Соглашения для агента

- **Минимальный diff** — не расширять scope без запроса
- **Следовать плану** — `план-цифровой-профиль.md` и `wireframe.md` — источник истины по структуре и дизайну
- **Стили** — только в `<style>` внутри `templates/base.html`, без внешних CSS-файлов
- **Без JS** — навигация, меню, smooth scroll через HTML/CSS
- **Доступность** — семантика (`header`, `main`, `section`, `nav`), `alt`, контраст, skip-link
- **Системные шрифты** — без CDN
- **Не коммитить** — `.env`, `node_modules/`, `dist/` (см. `.gitignore`)
- **Автокоммиты** — после каждого выполненного действия сразу commit; сообщение на усмотрение агента, но с описанием сделанного (см. `.cursor/skills/git-commits/`)
- **Не писать лишние markdown-файлы** — документация уже в перечисленных `.md`

## Ключевые документы

| Файл | Назначение |
