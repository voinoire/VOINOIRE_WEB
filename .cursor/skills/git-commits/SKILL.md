# Git-коммиты (цифровой профиль)

## Главное правило

**После каждого выполненного действия — сразу commit.** Не спрашивать подтверждение.

Если изменений нет — commit не нужен.

## Сообщение коммита

- **Язык:** русский
- **Формулировка:** на усмотрение агента — коротко и по сути
- **Обязательно:** явно указать, **что было сделано** (результат, не список файлов)
- **Без** префиксов `feat:` / `fix:` / conventional commits
- **Не** копировать шаблоны слепо — подбирать текст под конкретную работу

**Хорошо:**
- `перестроены секции шаблонов под цифровой профиль`
- `добавлен skill автокоммитов в .cursor/skills/`
- `placeholder SVG для аватара и карточки проекта`

**Плохо:**
- `update files`
- `fix`
- `изменения в templates/index.ru.html, base.html, build.py`

При крупной работе — вторая строка с деталями (опционально):

```
полировка hero и карточек хаба ссылок

hover, тени, адаптив под mobile first
```

## Workflow после действия

1. Убедиться, что задача завершена
2. Параллельно:
   ```bash
   git status
   git diff
   git log --oneline -5
   ```
3. Добавить только релевантные файлы; **никогда** не включать:
   - `.env`, credentials
   - `node_modules/`, `__pycache__/`, `.venv/`
   - `dist/` (генерируется локально, в `.gitignore`)
4. Сформулировать сообщение — что сделано
5. Commit:
   ```bash
   git add <релевантные файлы>
   git commit -m "$(cat <<'EOF'
   Сообщение: что было сделано.

   EOF
   )"
   git status
   ```
6. Кратко сообщить пользователю: hash или subject коммита

## Pre-commit checklist

- [ ] В staging только файлы текущей задачи
- [ ] `dist/` не попал в commit
- [ ] Если менялись шаблоны — `python build.py` проходит
- [ ] Сообщение описывает результат работы

## Запреты (без изменений)

- Не менять `git config`
- Не `--force`, `--hard`, `--no-verify` без явной просьбы
- Не `git commit --amend`, кроме случаев из user rules
- **Не push** без просьбы пользователя
- Не коммитить секреты

## Pull Request

PR — только по просьбе. Тогда `gh pr create` по user rules.

Test plan для PR:

- [ ] `python build.py` без ошибок
- [ ] RU: `/`, EN: `/en/`
- [ ] Якоря `#profile`, `#links`, `#project`, `#contact`
